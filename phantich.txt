. Input

Input là danh sách các quyển sách:

books = [
    {"id": 1, "title": "Python Basic", "quantity": 12},
    {"id": 2, "title": "FastAPI Beginner", "quantity": 3},
    {"id": 3, "title": "Clean Code", "quantity": 5},
    {"id": 4, "title": "Database Design", "quantity": 0},
    {"id": 5, "title": "Web API Design", "quantity": 20}
]

Ngoài ra có thể xuất hiện dữ liệu lỗi như:

{"id": 6, "title": "Java Basic"}

(thiếu trường quantity)

{"id": 7, "title": "Spring Boot", "quantity": -2}

(quantity âm)

2. Output

Nếu có sách sắp hết hàng:

{
    "message": "Danh sách sách sắp hết hàng",
    "data": [
        {
            "id": 2,
            "title": "FastAPI Beginner",
            "quantity": 3
        },
        {
            "id": 3,
            "title": "Clean Code",
            "quantity": 5
        },
        {
            "id": 4,
            "title": "Database Design",
            "quantity": 0
        }
    ]
}

Nếu không có sách nào:

{
    "message": "Không có sách nào sắp hết hàng",
    "data": []
}
3. Điều kiện xác định sách sắp hết hàng

Một quyển sách được xem là sắp hết hàng khi:

quantity <= 5

Điều kiện bổ sung:

Bỏ qua sách không có trường quantity.
Bỏ qua sách có quantity < 0.
Phần 2. Đề xuất 2 giải pháp
Giải pháp 1: Dùng vòng lặp for

Ý tưởng:

Tạo danh sách rỗng.
Duyệt từng sách.
Kiểm tra các điều kiện.
Nếu hợp lệ thì thêm vào danh sách kết quả.

Ví dụ:

result = []

for book in books:
    if "quantity" not in book:
        continue

    if book["quantity"] < 0:
        continue

    if book["quantity"] <= 5:
        result.append(book)
Ưu điểm
Dễ đọc.
Dễ thêm nhiều điều kiện.
Phù hợp với người mới học.
Nhược điểm
Code dài hơn.
Giải pháp 2: Dùng List Comprehension

Ví dụ:

result = [
    book
    for book in books
    if "quantity" in book
    and book["quantity"] >= 0
    and book["quantity"] <= 5
]
Ưu điểm
Code ngắn gọn.
Viết nhanh.
Nhược điểm
Khó đọc hơn khi có nhiều điều kiện.
Khó mở rộng khi cần xử lý logic phức tạp.
Phần 3. So sánh và lựa chọn giải pháp
Tiêu chí	Vòng lặp for	List comprehension
Độ dễ hiểu	Cao	Trung bình
Độ ngắn gọn	Trung bình	Cao
Dễ xử lý bẫy dữ liệu	Cao	Trung bình
Dễ bảo trì	Cao	Trung bình
Giải pháp được chọn

Chọn giải pháp dùng vòng lặp for.

Lý do
Dễ đọc và dễ hiểu.
Dễ bổ sung các điều kiện kiểm tra.
Phù hợp với API cần xử lý nhiều trường hợp dữ liệu lỗi.
Thuận tiện bảo trì khi hệ thống mở rộng.
Phần 4. Thiết kế các bước xử lý
Khởi tạo ứng dụng FastAPI.
Khai báo danh sách books.
Tạo endpoint GET /books/low-stock.
Tạo danh sách kết quả.
Duyệt từng sách trong books.
Nếu thiếu quantity thì bỏ qua.
Nếu quantity < 0 thì bỏ qua.
Nếu quantity <= 5 thì thêm vào danh sách kết quả.
Nếu danh sách kết quả rỗng thì trả về:
{
    "message": "Không có sách nào sắp hết hàng",
    "data": []
}
Nếu có dữ liệu thì trả về:
{
    "message": "Danh sách sách sắp hết hàng",
    "data": [...]
}
